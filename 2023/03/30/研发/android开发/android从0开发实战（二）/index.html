Android开发第二弹来啦~~
notes:标量值花括号警告
有个奇怪的地方要记录一下，我的c++源文件中使用了aotumic原子类型，于是调用了<aotumic.h>头文件并使用了其中定义的宏
#define ATOMIC_FLAG_INIT {false}
#define ATOMIC_VAR_INIT(__v) {__v}

看起来是不是没有任何问题，但是实际上编译的时候系统发出了警告，警告如下:
warning:braces around scalar initializer [-Wbraced-scalar-init]
查阅资料发现：

在C语言中，在初始化标量值时使用额外的大括号是完全合法的，如int x = { 5 };GCC会为此类代码生成警告。
它认为您可能写了一些您不想写的东西，因为花括号大部分时间用于启动聚合的多部分初始化程序，而不是用于标量的独立初始化程序。

这是一个标量初始化程序：int foo = 3;
这是一个带括号的标量初始化程序：int foo = {3};
这是数组的初始化器，不是标量：int foo[] = {1, 2, 3};
警告说明您的结构具有标量初始值设定项，并带有大括号

所以gcc的编译器检查的时候其实会更严格╮(╯▽╰)╭

虽然C源代码已经引用进去了并且编译过啦，但是怎么让jni调用C的函数并且确认接口被调用了呢，这就需要用到活动;
